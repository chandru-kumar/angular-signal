About effects

Use effect(() => { ... }) to run side-effects in response to signal changes. Effects run synchronously when their tracked dependencies change and should be used for tasks like logging, integration with non-Angular libraries, timers, or other external side-effects.

Example

// inside constructor or setup function
const count = signal(0);
const lastMessage = signal(null);

effect(() => {
  const v = count();
  lastMessage.set(`Effect observed count = ${v}`);
});

// update
count.update(n => n + 1);

See Angular Signals docs for guidance on effect cleanup and performance considerations.
