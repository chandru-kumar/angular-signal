Message Service

This component demonstrates a shared, application-wide message service backed by a readonly signal. Use the textbox above to show a message. The MessageService exposes a readonly signal message and helper methods show(text, durationMs?) and clear().

Why a signal-based service?

  • Signals provide a simple reactive source that any component can read.
  • Using a readonly view prevents consumers from mutating internal state.
  • Services can encapsulate auto-clear behavior and formatting concerns.

Example

messageService.show('Saved successfully', 3000);
// in component template
<app-message></app-message>
